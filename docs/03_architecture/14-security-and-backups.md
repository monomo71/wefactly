# Security and Backups

## Basisprincipe
Veiligheid is geen feature, maar een vereiste.

## Security regels
- veilige login
- sessiebeheer
- input validatie
- geen gevoelige data in frontend
- secrets buiten code (.env)

## Configuratie
- environment variables
- geen hardcoded credentials
- duidelijke setup

## Self-hosted veiligheid
- veilige defaults
- admin setup flow
- duidelijke documentatie

## Backups
Minimaal:
- database export
- restore mogelijkheid
- testbare recovery

## SaaS (later)
- tenant isolatie
- betere auth
- logging
- monitoring
- disaster recovery

## Belangrijk
> Alles moet herstelbaar zijn.

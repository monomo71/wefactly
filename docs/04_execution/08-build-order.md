# Build Order

## Doel
Gefaseerd bouwen om:
- focus te behouden
- chaos te voorkomen
- herhaling van eerdere fouten te vermijden

## Fase 1 — Fundament
- Project setup (Vite + React)
- Basis structuur
- Routing
- Auth (basic)
- Layout (sidebar + header)
- Design system basis
- Environment setup

## Fase 2 — Klanten
- Klantenlijst
- Klant toevoegen/bewerken
- Adresstructuur
- Validatie

## Fase 3 — Producten
- Productlijst
- Product toevoegen/bewerken
- Prijzen
- Basis velden

## Fase 4 — Offertes
- Offerte maken
- Regels toevoegen
- Opslaan als concept
- Status

## Fase 5 — Facturen
- Factuur maken
- Regels
- Statusmodel (draft/sent/paid/overdue)
- Nummering

## Fase 6 — PDF
- Factuur PDF
- Offerte PDF
- Layout en styling

## Fase 7 — Mail
- Versturen via mail
- Templates
- Instellingen

## Fase 8 — Dashboard
- Info tegels
- Snelle acties
- Recente items

## Fase 9 — Afwerking
- Error handling
- Loading states
- Empty states
- Tooltips
- UX polish

## Regel
> Bouw alleen wat in de huidige fase hoort.
